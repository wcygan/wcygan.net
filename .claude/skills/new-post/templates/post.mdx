One or two sentences that state the subject directly. No "in this post" or "let's dive into." Open with the thing.

## First section

Body paragraphs. Keep sentences short. State opinions flat.

Before/after, code, or a concrete artifact belongs near the claim it supports:

```bash
→ some-tool --flag input
expected output
```

### A subsection when you need one

Use `###` sparingly. Don't nest deeper.

## A section that earns a diagram

{/_ Diagrams are static SVGs built from Mermaid source, not a runtime component. 1. Put the Mermaid text in src/diagrams/<this-post-slug>/<name>.mmd
(add accTitle + accDescr lines inside the source for accessibility). 2. Run `deno task render:diagrams` to compile it to public/<slug>/<name>.svg
(the width/height it prints go on the <img> below). 3. Embed the result like this (use `static-mermaid-frame-simple` for small ones): _/}

<figure className="static-mermaid-figure">
  <div className="static-mermaid-frame">
    <img
      className="static-mermaid-diagram"
      src="/example-post-title/some-diagram.svg"
      alt="One-sentence description of what this diagram shows."
      width={400}
      height={300}
    />
  </div>
  <figcaption>Short caption describing the diagram.</figcaption>
</figure>

The diagram replaces a paragraph — don't also write the paragraph.

## Optional pull-quote

At most one per post, for a line that should land alone:

<aside class="pull-quote">
  <p>The standout line goes here — one sentence, no attribution.</p>
</aside>

## Closing

Short. A shrug or a concrete line. **Never `## Conclusion`.**
