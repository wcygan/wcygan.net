How to keep your derived data source (e.g., [OpenSearch](https://github.com/opensearch-project/opensearch), or some data lake) in sync with the source of truth (e.g., [Postgres](https://github.com/postgres/postgres))?

The answer: [Change Data Capture](https://en.wikipedia.org/wiki/Change_data_capture)

```sql
UPDATE users
SET plan = 'pro'
WHERE id = 42;
```

Once the row is [committed](https://www.postgresql.org/docs/current/sql-commit.html), a new
[WAL entry](https://www.postgresql.org/docs/current/wal-intro.html) exists in
Postgres. [Debezium](https://github.com/debezium/debezium) [reads that entry](https://debezium.io/documentation/reference/stable/connectors/postgresql.html#postgresql-events) and [publishes the change to Kafka](https://kafka.apache.org/42/getting-started/introduction/#what-is-event-streaming). Next, a [Kafka Consumer](https://www.baeldung.com/java-kafka-consumer-api-read#2-consuming-messages) (the _indexer_) reads & processes each message and [updates the document in OpenSearch](https://docs.opensearch.org/latest/api-reference/document-apis/update-document/).

Finally, the next client to [fetch the document from OpenSearch](https://docs.opensearch.org/latest/api-reference/document-apis/get-documents/) reads the correct value (`Plan = Pro`)
