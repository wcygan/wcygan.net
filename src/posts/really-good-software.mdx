## LazyGit

[https://github.com/jesseduffield/lazygit](https://github.com/jesseduffield/lazygit)

<aside class="pull-quote">
  <p>simple terminal UI for git commands</p>
</aside>

Before LazyGit, working on stacked PRs was a pain...

```bash
git checkout main
git pull --rebase
git checkout wcygan/branch-one
git merge main
git checkout wcygan/branch-two
git merge wcygan/branch-one
git checkout wcygan/branch-three
git merge wcygan/branch-two
...
```

After LazyGit, the amount of keystrokes that I needed to achieve this went down by ~80%!

## fzf

[https://github.com/junegunn/fzf](https://github.com/junegunn/fzf)

<aside class="pull-quote">
  <p>fzf is a general-purpose command-line fuzzy finder.</p>

I use this to fuzzy-search my `history`. It's similar to doing this:


This makes it really easy to find commands you did in the past.

## uv

[https://docs.astral.sh/uv/](https://docs.astral.sh/uv/)

<aside class="pull-quote">
  <p>An extremely fast Python package and project manager, written in Rust.</p>
</aside>

`uv` has become my defacto scripting platform. With [PEP 723 inline script metadata](https://docs.astral.sh/uv/guides/scripts/), a single `.py` file can declare its own dependencies and `uv run` will resolve them into an ephemeral environment before executing.

```python
# /// script
# requires-python = ">=3.12"
# dependencies = ["httpx"]
# ///
import httpx
print(httpx.get("https://example.com").status_code)
```

```bash
→ uv run script.py
200
```

## just

[https://github.com/casey/just](https://github.com/casey/just)

<aside class="pull-quote">
  <p>just is a handy way to save and run project-specific commands.</p>
</aside>

I use `just` for task orchestration across every project — it's a `make` replacement that doesn't pretend to be a build system. Recipes are readable, arguments are first-class, and a `justfile` at the repo root becomes the one obvious place to find "how do I run this thing."

## jj (experimental)

[https://github.com/jj-vcs/jj](https://github.com/jj-vcs/jj)

<aside class="pull-quote">
  <p>
    Jujutsu is a powerful version control system for software projects. You use
    it to get a copy of your code, track changes to the code, and finally
    publish those changes for others to see and use.
  </p>
</aside>

I'm currently testing `jj` as a replacement for `git`. It's git-compatible at the wire level — my coworkers don't need to know — but local history rewriting, rebasing, and conflict handling feel dramatically better. Still early days for me, but the ergonomics of `jj split`, `jj squash`, and first-class conflict objects are hard to go back from.
