The [Ghostty](https://ghostty.org/) animation is really cool, so I animated a [Hot Dog](https://poly.pizza/m/eiPR4iwcYpa) (and other things).

## How it works

The pipeline goes from a 3D model to a rotating ASCII flipbook in four stages:

1. **Start with a 3D model.** The source is a `.glb` file (which I found on [Poly Pizza](https://poly.pizza/)).
2. **Spin it and take photos.** [Blender](https://www.blender.org/) loads the model, points a camera at it, and saves 60 to 120 images as it rotates a full 360°. Each image is one frame of the spin.
3. **Turn each photo into ASCII art.** A Python script scans each image in a grid. For every cell it picks a character and tags it with a color. All the frames are saved together in one file.
4. **Flipbook it in the browser.** The page loads that file and swaps which frame is on screen about 30 times a second.

## In practice

The example can be found at [wcygan/rotating-animations](https://github.com/wcygan/rotating-animations).

Render 60 frames of a model spinning in Blender — see [`scripts/render_spin.py`](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/scripts/render_spin.py):

```bash
blender -b -P scripts/render_spin.py -- models/penguin.glb frames_penguin 60 upright
```

Convert that frame directory into an ASCII flipbook JSON — see [`scripts/to_ascii_generic.py`](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/scripts/to_ascii_generic.py):

```bash
uv run scripts/to_ascii_generic.py frames_penguin web/src/penguin.json
```

The converter uses [Pillow](https://pillow.readthedocs.io/) (a Python image library) to read each PNG and scan it in a grid (100 columns × 41 rows by default). The [character ramp](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/scripts/to_ascii_generic.py#L28) goes from sparse to dense, and the [per-cell loop](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/scripts/to_ascii_generic.py#L60-L99) averages the pixels in each cell, picks a character based on how filled-in that area is, and picks a color group based on how vivid the color is. Neighboring cells with the same color group get merged into one HTML element, and all 60 frames end up in a [single JSON file](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/web/src/penguin.json) the page imports.

Per-model converters (e.g. [`to_ascii_mallard.py`](https://github.com/wcygan/rotating-animations/blob/7a9f9b4c38d9f10dc4a1dadb46962afde862c6f8/scripts/to_ascii_mallard.py)) follow the same shape but swap in hand-tuned color thresholds, so different parts of the model (body, head, bill, feet, collar) get their own distinct colors.
